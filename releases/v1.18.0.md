# Release Notes - v1.18.0

**Release Date:** June 2026
**TIBCO Platform Control Plane Version:** 1.18.0
**Platform:** Amazon Elastic Kubernetes Service (EKS)
**Status:** Current release

## Overview

This release updates the EKS workshop documentation for TIBCO Platform Control Plane 1.18.0. It carries forward the 1.17.0 simplified DNS model and adds 1.18.0 Control Plane features that affect EKS operators: Gateway API controller support, namespace-level RBAC, Alert Audit Trail, Console-managed email server configuration, and Developer Hub self-service flows.

Official documentation:

- [TIBCO Platform Control Plane 1.18.0 Documentation](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Default.htm)
- [Official 1.18.0 New Features](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Release-Notes/new-features.htm)
- [Official 1.18.0 Known Issues](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Release-Notes/known-issues.htm)

## What's New in v1.18.0

### TIBCO Control Plane

#### Gateway API Controller Support

Gateway API Controller is supported for Control Plane and Control Tower data planes. On EKS, keep the existing AWS ALB plus Nginx or Traefik ingress pattern for the default workshop route, and evaluate Gateway API where the Control Plane, capability, and Data Plane are ready for `Gateway` and `HTTPRoute` resources. The EKS guide now includes an NGINX Gateway Fabric 2.3.0 example and a `tibco-cp-base` `gatewayRoute` override for Control Plane router and hybrid-proxy traffic.

#### Namespace-Level RBAC for Data Planes

Kubernetes namespaces are now Data Plane-level resources managed under Data Plane Configuration > Resources. Application Manager and Application Viewer role assignment screens include Capability and Namespace selectors for granular access control.

#### Email Server Configuration in Platform Console

Email server configuration moved from `tibco-cp-base` chart values to the TIBCO Platform Console UI. For EKS, configure SES, SMTP, SendGrid, or Microsoft Graph from the Console after installation or upgrade. Remove deprecated email Helm values before upgrading to 1.18.0.

Deprecated values removed by the 1.18 upgrade assistant include:

- `global.external.emailServerType`
- `global.external.fromAndReplyToEmailAddress`
- `global.external.cronJobReportsEmailAlias`
- `global.external.platformEmailNotificationCcAddresses`
- `global.external.emailServer`

#### Alert Viewer and Alerts Audit Trail

The UI includes an Alerts Audit Trail page that records system health and rule performance events.

#### Developer Hub Self-Service Flows

Developer Hub self-service flows provide reusable building blocks for platform automation through customizable templates and platform APIs.

### Capability Updates

- BW5 and BW6 Containers add Traefik Gateway API support and an `Other` Gateway API controller option.
- Flogo adds Traefik Gateway API support, NetScaler Gateway support at the Data Plane level, and namespace-level RBAC enforcement.
- Hawk 1.18.12 charts are available for the 1.18 release stream.

## EKS-Specific Considerations

### Simplified DNS Continues

The simplified DNS model remains applicable in 1.18.0. EKS installations should prefer one Route 53 base domain such as `platform.aws.example.com`, with Platform Console at `admin.platform.aws.example.com` and subscriptions at `<hostPrefix>.platform.aws.example.com`. In Helm values, set `global.external.dnsDomain` and `global.external.dnsTunnelDomain` to the same base domain. The `hybrid-proxy` chart routes tunnel traffic through `/infra/tunnel` on that shared domain.

Use one wildcard Route 53 alias record and one ACM wildcard certificate for `*.platform.aws.example.com`. Keep separate `cp1-my` and `cp1-tunnel` domains only for legacy or explicitly separated deployments.

### Amazon Aurora PostgreSQL SSL

Most EKS production deployments use Amazon Aurora PostgreSQL. If the Aurora parameter group enforces `rds.force_ssl=1`, set `TP_DB_SSL_MODE=require` or `TP_DB_SSL_MODE=verify-full` before generating `tibco-cp-base` values. Use `verify-full` with the AWS RDS CA bundle when certificate verification is required.

### Gateway API Controllers

NGINX Gateway Fabric 2.3.0, Traefik `traefik-39.0.7`, Istio 1.28.2, and NetScaler `netscaler-cpx-with-gateway-controller-2.0.0` are listed in the official Gateway API controller compatibility matrix. Keep classic ALB plus Nginx or Traefik ingress for existing routes unless you intentionally migrate endpoint exposure to Gateway API resources.

### Namespace-Level RBAC

Before upgrading shared EKS data planes, inventory application namespaces and review Application Manager/Application Viewer assignments. After upgrade, users may need explicit namespace grants to deploy or view applications.

### Email with SES, SMTP, SendGrid, or Microsoft Graph

Do not pass email server settings in 1.18.0 Helm values. After installation or upgrade, configure SES, SMTP, SendGrid, or Microsoft Graph in the TIBCO Platform Console. Ensure EKS security groups, network policies, IAM permissions, and any Microsoft Entra application prerequisites allow the chosen email provider.

## Upgrade Path from v1.17.0

1. Back up Helm values, Kubernetes secrets, and Aurora PostgreSQL data.
2. Update the `tibco-platform` Helm repository.
3. Remove deprecated email Helm values from `tibco-cp-base` values.
4. Upgrade Control Plane base and infrastructure charts to 1.18.0-compatible versions.
5. Upgrade Data Plane components and capability charts.
6. Configure email server from the TIBCO Platform Console.
7. Review namespace-level RBAC assignments.
8. Validate Gateway API endpoint behavior if used.
9. Confirm Alerts Audit Trail events are recorded.

## Upgrade Path from v1.16.0

Upgrade to 1.17.0 first, then upgrade to 1.18.0:

1. Follow [v1.16.0 to v1.17.0 upgrade notes](./v1.17.0#upgrade-path).
2. Apply the v1.18.0 upgrade path above.

## Known Issues to Review

The official 1.18.0 known issues include items relevant to EKS and Traefik users:

- Existing 1.17.0 Data Planes can show incorrect Integration Summary and Application Instance card data after Control Plane upgrade until the Data Plane is upgraded to 1.18.0.
- Traces and application service metrics have known limitations with Traefik in some Control Tower data plane environments.
- BW6 Swagger tester behavior can require BW6 capability and application upgrades when changing endpoint visibility through Traefik Gateway.
- Flogo capability provisioning can fail if users navigate backward and forward in early wizard steps; restart the provisioning flow as a workaround.

Review the complete official list before production upgrades: [Official 1.18.0 Known Issues](https://docs.tibco.com/pub/platform-cp/1.18.0/doc/html/Release-Notes/known-issues.htm).
